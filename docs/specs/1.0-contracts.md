# Pipeline Compose 1.0 contracts

This document freezes the public contracts shipped in **v1.0.0**. Patch releases (1.0.x) may add validation or docs but must not break these guarantees without a semver major.

## Pipeline document (schema v2)

| Field | Contract |
|-------|----------|
| `version: 2` | Required. Schema v1 documents are rejected. |
| `pipelines` | Map of named pipeline definitions merged at resolve time. |
| `groups` / `group` | Optional grouping metadata; stages inherit or override `group`. |
| `companion_workflows` | Optional allowlist of workflow paths excluded from orphan warnings. |
| Stage `id` | Unique within each pipeline definition; stable orchestration key. |
| Stage `workflow` | Repo-relative path to a callable/dispatchable workflow file. |
| Stage `needs` | DAG edges; unknown stage ids are validation errors (`needs.unknown`). |
| Stage `when` | Expression evaluated against `github` + `context` (same language as eval action). |
| Stage `repo` | Optional `owner/repo` for cross-repo dispatch; requires repo token map or GitHub App credentials at run time. |
| Stage `inputs` / `outputs` | Context merge contract for downstream stages. |

## CLI commands

| Command | Contract |
|---------|----------|
| `compile` | Emits orchestrator workflow YAML; default compile action pin `@v1.4.0`. |
| `validate` | Non-destructive report; `--strict` promotes warnings to errors; `--workflows` checks files + orphans. |
| `validate --mermaid` | Renders topology; error/blocked styling from validation issues. |
| `init` | Scans `.github/workflows` and writes v2 pipeline YAML. |
| `sync` | Copies group-prefixed workflows per sync plan. |
| `eval` | Evaluates `when:` expressions with `--expression`, `--context`, `--github`. |

## Validation issue codes (stable)

Errors in `--strict` mode fail the process (exit 1):

- `workflow.missing`, `workflow.orphan` (orphan is warn unless strict)
- `stage.repo-invalid`, `needs.unknown`
- Deprecation codes promoted in strict: `export.manual-upload-deprecated`, `uses.monorepo-subpath-deprecated`, `uses.master-pin-deprecated`, `pipeline.v1-deprecated`

Warnings (non-strict): `group.path-prefix`, `group.mixed`, `stage.cross-repo`, `stage.cross-repo-token`, `export.missing`.

## Actions (published packages)

| Action | Role |
|--------|------|
| `pipeline-compose-run` | Orchestrates stages, context merge, cross-repo token mapping or GitHub App auth. |
| `pipeline-compose-compile` | Compile pipeline YAML to workflow in CI. |
| `pipeline-compose-eval` | Expression evaluation in custom workflows. |
| `pipeline-compose-export` | Artifact upload contract `pipeline-compose-<stage_id>` / `outputs.json`. |
| `pipeline-compose-context-merge` | Manual context file updates. |

Action repos publish immutable semver tags from this monorepo; consumers pin `@v1.4.0` or newer semver tags.

## CI guarantees (meta repo)

- Unit tests + coverage on every push/PR.
- Strict validate for meta pipeline and all `examples/*` pipelines with `--workflows`.
- Compile action parity: CLI output matches action-compile output for meta pipeline.
- PR bot posts validation mermaid + issues on pipeline schema changes.

## Shipped in 1.1.x

- **`validate --simulate`** — dry-run stage table (`when:`, upstream skips, missing context).

## Shipped in 1.2.x

- **Parallel stage dispatch** — run orchestrator dispatches independent stages (same DAG wave) concurrently.
- **Pipeline `concurrency`** — optional root field; run action enforces on entry workflow; compile emits native block.

## Shipped in 1.3.x

- **Context input validation** — `context.unknown-stage` / `context.unknown-output` on invalid `inputs` wiring.
- **Simulate wave grouping** — `--simulate` output groups stages by parallel DAG wave.

## Shipped in 1.4.x

- **Smart rerun** — `smart_rerun: true` on pipeline YAML; workflow re-run reuses prior attempt stage outputs when inputs are unchanged.
- **Sub-pipelines** — `pipeline_file` stages nest another pipeline inline (one level deep).
- **Typed context** — `context_schema` JSON Schema validates output declarations and `context.*` wiring.

## Shipped in 1.5.x

- **Stage catalog** — root `catalog` map of reusable stage templates; stages use `use: <key>` with optional overrides.

## Shipped in 1.6.x

- **GitHub App cross-repo auth** — run action can mint installation tokens via `github_app_id` + `github_app_private_key` when `repo_tokens_json` does not include a target slug.
